# 智慧大棚项目 - AI 协作规范

## 项目信息
- 名称：智慧大棚多模态测控与病虫害边缘预警系统
- 作者：陈德源（梧州学院 23通信工程）
- 用途：大创项目 + 毕业设计

## 硬件架构
| 模块 | 型号 | STM32接口 | 备注 |
|------|------|-----------|------|
| 温湿度 | SHT30 | I2C1 (PB6/PB7) | 地址 0x44 |
| 光照 | BH1750 | I2C1 (PB6/PB7) | 地址 0x23，与SHT30共享I2C1 |
| 土壤湿度 | 电容式 | ADC1_CH0 (PA0) | - |
| OLED | 0.96寸 SSD1306 | 模拟I2C (PB12/PB13) | 地址 0x78 |
| K210 | DNK210 + OV2640 | UART2 (PA2=TX, PA3=RX) | J3排针 IO6/IO8 |
| ESP8266 | ESP-01S | UART3 (PB10=TX, PB11=RX) | AT固件 v2.2+ |
| 继电器 | 4路 5V 光耦 | PA4(风机),PA5(水阀),PA6(补光灯) | 高电平使能 |
| 蜂鸣器 | 有源 | PA8 | - |
| 风扇 | 直流电机 PWM | PA1 (TIM2_CH2) | - |

## 编码规范（关键！）
- **所有 .c/.h 文件必须 GBK 编码**
- ARMCC v5 编译器不支持 UTF-8
- OLED 中文字模使用 GBK 编码的 16x16 点阵
- `OLED_ShowChinese` 步长为 2（GBK），不是 3（UTF-8）
- 串口 printf 中文也会以 GBK 输出

## 软件架构
- **调度器**: SysTick 1ms 计数 + while(1) 轮询任务列表
- **标准外设库**: STM32F10x_StdPeriph_Lib（非 HAL）
- **I2C**: 硬件 I2C1 (100kHz)，SHT30/BH1750 共享
- **UART2**: 环形缓冲区 64 字节，接收 K210 4 字节帧
- **K210 帧协议**: [0xAA][类别 1B][置信度 1B][校验和 1B]
  - 类别: 0x01=早疫病 0x02=晚疫病 0x03=叶斑病 0x04=健康

## 关键踩坑记录
1. SHT30 I2C 延时需 `j<6000` 才能达到约 1ms（72MHz -O0）
2. 初始顺序必须 UART2→SHT30，否则 SHT30 失败
3. OLED 用独立模拟 I2C (PB12/PB13)，不占用硬件 I2C1
4. BH1750 和 SHT30 共享 I2C1 时，BH1750 不重复初始化
5. main.c 备份版是 UTF-8 编码，中文正常版是 GBK——必须用 GBK
